## 一、项目概述

### 1.1 项目简介

**QuantumTrader AI** 是一个开源的AI驱动量化交易系统，专注于加密货币交易。系统采用前后端分离架构，集成了多种经典交易策略、AI分析能力、风控系统和实时数据推送。

### 1.2 核心定位

- **开源AI量化交易系统** - 面向开发者和量化爱好者
- **多交易所支持** - 目前支持欧易（OKX），架构可扩展
- **多策略框架** - 内置6种经典策略，可自定义扩展
- **AI驱动** - 集成DeepSeek AI，支持行情分析、策略解释
- **实时推送** - WebSocket实时行情、订单、持仓更新
- **风控优先** - 三层风控体系，资金安全第一

### 1.3 技术栈

**前端：**
- Vue 3 + TypeScript
- Vite 构建工具
- Pinia 状态管理
- Element Plus UI组件库
- ECharts 图表库
- Vue Router 路由
- TailwindCSS 样式框架

**后端：**
- Python 3.12+
- FastAPI 高性能异步框架
- SQLAlchemy ORM
- SQLite 数据库（可扩展为PostgreSQL）
- JWT 认证
- WebSocket 实时通信
- AES-256 加密（Fernet）

**交易所：**
- 欧易（OKX）模拟盘优先
- 支持现货交易
- 支持期权交易（模拟数据）

**AI能力：**
- DeepSeek API（云端）
- llama.cpp（本地，开发中）
- 混合调度架构

**通知：**
- PushPlus 微信推送

### 1.4 项目特点和优势

1. **完整的交易功能** - 下单、撤单、查持仓、查余额一应俱全
2. **6种经典策略** - 均线交叉、网格、RSI、MACD、布林带、定投
3. **三级风控体系** - 系统级+策略级+单交易级，熔断机制
4. **实时数据推送** - WebSocket架构，行情/订单/持仓实时更新
5. **专业图表组件** - 6个ECharts图表，K线支持4种指标
6. **AI智能分析** - 行情分析、策略解释、AI对话
7. **期权交易支持** - T型报价、希腊字母、完整交互流程
8. **响应式设计** - 支持PC、平板、手机多端
9. **API密钥加密存储** - AES-256加密，脱敏显示
10. **开源友好** - 代码规范，模块化设计，易于扩展

## 四、代码规范和开发要求

### 4.1 前端代码规范

**Vue 3 + TypeScript：**
- 使用Composition API（`<script setup>`语法）
- 优先使用TypeScript类型定义
- 组件名使用PascalCase
- props必须定义类型和默认值
- emit必须定义类型

**命名规范：**
- 组件名：PascalCase（如 `KLineChart.vue`）
- 组合式函数：camelCase，use开头（如 `useWebSocket`）
- 变量名：camelCase
- 常量名：UPPER_SNAKE_CASE
- 文件名：组件用PascalCase，其他用kebab-case

**样式规范：**
- 优先使用TailwindCSS
- 自定义样式写在`<style scoped>`中
- 避免使用!important
- 颜色使用CSS变量或Tailwind主题色

**注释要求：**
- 复杂逻辑必须加注释
- 组件顶部要有功能说明
- 公共函数必须有JSDoc注释
- TODO注释要标注日期和负责人

### 4.2 后端代码规范

**Python + FastAPI：**
- 使用类型注解（Type Hints）
- 遵循PEP 8规范
- 使用async/await异步编程
- 依赖注入使用Depends

**命名规范：**
- 类名：PascalCase
- 函数名：snake_case
- 变量名：snake_case
- 常量名：UPPER_SNAKE_CASE
- 模块名：snake_case

**API设计规范：**
- RESTful风格
- 统一响应格式：`{code: 0, message: "success", data: ...}`
- 错误使用HTTP状态码 + 业务错误码
- 分页参数：page, page_size
- 分页响应：`{items: [], total: 0, page: 1, page_size: 20}`

**数据库规范：**
- 表名：snake_case，复数形式（如 `users`）
- 字段名：snake_case
- 主键：id
- 时间戳：created_at, updated_at
- 软删除：deleted_at

**注释要求：**
- 模块顶部要有模块说明
- 类和公共方法要有docstring
- 复杂算法要有步骤说明
- TODO注释要标注

### 4.3 Git提交规范

**提交信息格式：**
```
<type>(<scope>): <subject>

<body>

<footer>
```

**type类型：**
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具相关

**示例：**
```
feat(strategy): 添加RSI超买超卖策略

- 实现RSI指标计算
- 添加策略回测支持
- 添加实盘运行支持

Closes #123
```

### 4.4 安全规范

1. **API密钥安全**
   - 绝对不能明文存储API密钥
   - 必须使用AES-256加密
   - 前端显示脱敏（前4后4）
   - 日志中不能打印完整密钥

2. **用户密码安全**
   - 使用bcrypt哈希存储
   - 密码强度校验
   - 防止SQL注入

3. **交易安全**
   - 测试必须用模拟盘
   - 风控检查不可绕过
   - 重要操作二次确认

4. **输入验证**
   - 所有用户输入必须验证
   - 防止XSS攻击
   - 防止CSRF攻击

- Token是否有效
- 网络是否通畅

**Q: 策略不运行？**
A: 请检查：
- 策略状态是否为"运行中"
- API密钥是否配置
- 风控是否触发暂停
- 检查策略运行日志

## 八、安全要求

### 8.1 API密钥加密存储

**必须遵守：**
1. 所有API密钥必须加密存储，不能明文
2. 使用AES-256加密算法（Fernet）
3. 加密密钥不能硬编码在代码中
4. 加密密钥通过环境变量配置

**加密密钥要求：**
- 32个url-safe base64编码的字节
- 生产环境必须修改默认密钥
- 密钥丢失将无法恢复已加密的API密钥

### 8.2 脱敏显示

**前端显示：**
- API Key只显示前4位和后4位，中间用*代替
- 示例：`abcd****1234`

**日志输出：**
- 日志中不能打印完整的API密钥
- 日志中不能打印用户密码
- 敏感信息必须脱敏

### 8.3 风控要求

**强制风控：**
1. 每笔交易必须经过风控检查
2. 风控检查不可绕过
3. 触发熔断后必须暂停交易
4. 用户可以调整风控参数，但不能关闭风控

**三级熔断：**
- 一级熔断（5%）：仓位降50%，只减仓不加仓
- 二级熔断（10%）：暂停开新仓，只能平仓
- 三级熔断（15%）：全部平仓，系统暂停

### 8.4 安全最佳实践

1. **定期更换API密钥**
2. **使用IP白名单限制API访问**
3. **开启提现白名单**
4. **不要在公共设备上登录**
5. **定期检查账户安全**
6. **开启二次验证（2FA）**
7. **定期备份数据**
8. **及时更新系统版本**
