# QuantumTrader AI 量化交易系统 - 后续开发指南

## 项目概述

**QuantumTrader AI** 是一套全自动AI量化交易系统，采用科幻交易终端风格UI，支持多交易所、三大交易品类（现货、合约、期权）、AI驱动策略、实盘+模拟盘双轨制。

### 当前状态
- **整体完成度：82%**
- 后端API：63个接口
- 前端组件：20+个
- 策略数量：6个
- 图表组件：6个

### 技术栈
- **后端**：Python 3.12+、FastAPI、SQLite + SQLAlchemy、JWT、WebSocket
- **前端**：Vue 3 + TypeScript、ECharts、TailwindCSS
- **AI**：DeepSeek API + 本地llama.cpp
- **交易所**：欧易OKX（模拟盘+实盘）
- **通知**：PushPlus

**目标**：真正跑通本地大模型推理，实现零成本、隐私保护的AI能力。

- 流式输出支持（打字机效果）
- 推理速度统计（tokens/s）
- 内存占用监控

##### 3.2 GPU加速支持（可选，有时间就做）
- NVIDIA GPU：CUDA加速
- Apple Silicon：Metal加速
- 自动检测硬件，选择最优配置
- 可手动切换CPU/GPU

##### 3.3 模型管理
- 模型列表页面：
  - 显示已下载的模型
  - 每个模型显示：名称、参数量、量化等级、文件大小、状态
  - 当前默认模型标识
  - 操作按钮：设为默认、加载、卸载、删除、查看详情

- 模型下载：
  - 推荐模型列表（官方精选）
  - 每个模型显示：参数、量化等级、推荐用途、下载大小
  - 一键下载
  - 下载进度条显示
  - 断点续传支持
  - 下载完成MD5校验

- 模型导入：
  - 支持上传本地GGUF文件
  - 自动解析模型信息（参数量、量化等级）
  - 导入进度显示
  - 导入后加入模型列表

##### 3.4 模型加载/卸载
- 加载模型到内存
- 显示加载进度
- 卸载模型释放内存
- 智能卸载：长时间不用自动卸载
- 可配置自动卸载时间

##### 3.5 云端+本地混合调度
- 智能路由：
  - 简单任务（数据整理、格式转换）自动用本地模型

  - 本地模型加载失败 → 自动用云端
  - 云端API调用失败 → 自动用本地模型
  - 都不可用 → 友好提示，不崩溃

- 费用统计：
  - 统计本地模型节省了多少Token费用
  - 显示"已节省 $XX.XX"

#### 4. 策略运行状态持久化测试验证

**目标**：确保服务重启后策略能正确恢复，没有Bug。

**具体任务：**

##### 4.1 功能测试
- 启动一个策略 → 重启服务 → 检查策略是否自动恢复运行
- 启动多个策略 → 重启服务 → 检查是否全部恢复
- 暂停一个策略 → 重启服务 → 检查是否保持暂停状态
- 停止一个策略 → 重启服务 → 检查是否保持停止状态
- 策略运行中触发风控暂停 → 重启服务 → 检查状态是否正确
- 异常退出（模拟崩溃）→ 重启服务 → 检查策略状态

##### 4.2 数据一致性测试
- 重启前后策略收益是否一致
- 重启前后持仓是否一致
- 重启前后运行日志是否连续
- 重启前后订单是否正确关联

##### 4.3 异常处理
- 恢复失败的策略自动标记为stopped
- 记录恢复失败的原因
- 不影响其他策略的恢复
- 有错误日志

##### 4.4 边界情况
- 服务启动时数据库损坏怎么办
- 服务启动时交易所连不上怎么办
- 策略恢复时参数不兼容怎么办

**验收标准：**
- [ ] 服务重启后运行中的策略自动恢复
- [ ] 暂停/停止状态正确保持
- [ ] 数据一致性（收益、持仓、日志）
- [ ] 异常情况有处理，不会导致启动失败
- [ ] 恢复失败有日志记录

## 二、开发要求

### 2.1 代码质量
- 模块化设计，职责清晰
- 注释完善，关键逻辑必须有注释
- 命名规范，语义化
- TypeScript类型完整
- Python类型提示完整
- 错误处理完善，不静默失败

### 2.2 性能要求
- 首屏加载 < 3秒

- 滚动流畅 60fps
- WebSocket消息延迟 < 100ms
- 长列表（1000条）不卡顿

- API密钥AES-256加密存储
- 敏感数据脱敏显示

- JWT Token认证
- 输入验证和SQL注入防护
- XSS防护
- CSRF防护

### 2.4 用户体验
- 加载状态清晰（骨架屏、loading）
- 错误提示友好（用户能看懂）
- 操作有反馈（按钮状态、成功提示）
- 动画流畅（不卡顿、不闪白）
- 响应式适配（全设备）
- 无障碍支持（键盘操作、屏幕阅读器）

### 2.5 交易安全
- 所有交易用模拟盘测试
- 风控检查在下单前执行，不可绕过
- 危险操作二次确认

## 五、启动方式

### 后端启动
```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 前端启动
```bash
cd frontend
npm install
npm run dev
```

## 六、核心原则（必须遵守）

1. **数据真实原则**：所有交易数据、盈亏100%真实，严禁造假、美化、虚假展示
2. **功能完成原则**：未完成功能必须标注"开发中"，禁止假界面、文字游戏
3. **风险透明原则**：所有风险如实展示，不得隐瞒爆仓、强平、流动性风险
4. **代码可运行原则**：每一行代码必须可执行可测试，禁止空气代码
5. **稳定优先原则**：稳定性优先级高于一切新功能
6. **安全第一原则**：所有交易测试用模拟盘，绝对不要碰真实盘
7. **开源友好原则**：代码规范、注释完善，为后续开源做准备
